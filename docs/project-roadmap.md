## 2. 阶段一：资料与理论基底

目标是回答“直升机怎么建模、VRS 怎么进入模型、拿什么验证”。

已形成的资料包括：

- [阶段一资料地图](../一、理论基础与文献调研/一、阶段一资料地图.md)：把课程资料、教材、VRS 专题和后续建模资料串起来。
- [旋翼空气动力学基础](../一、理论基础与文献调研/二、旋翼空气动力学基础.md)：整理动量理论、叶素理论、挥舞和动态入流。
- [涡环状态研究综述](../一、理论基础与文献调研/三、涡环状态研究综述.md)：整理 VRS 流态、边界、判据、入流修正和验证方法。
- [建模依据与技术路线](../一、理论基础与文献调研/四、建模依据与技术路线.md)：整理建模对象、状态量、外力外矩和模块路线。
- [数据来源与验证依据](../一、理论基础与文献调研/五、数据来源与验证依据.md)：说明悬停、正常下降、VRS 进入、改出和参数扫掠怎么验证。
- [深度阅读索引](../一、理论基础与文献调研/九、深度阅读笔记/深度阅读索引.md)：跟踪已读和待读资料。
- [Wayne Johnson VRS 模型笔记](../一、理论基础与文献调研/八、专题文献笔记/Wayne%20Johnson涡环状态模型笔记.md)：给 VRS 平均入流和稳定边界修正提供依据。
- [动态入流模型笔记](../一、理论基础与文献调研/八、专题文献笔记/动态入流模型笔记.md)：说明为什么第一版先用均匀入流，再考虑 Pitt-Peters。

阶段一的完成标准：

- 能说清模型对象是常规单主旋翼带尾桨直升机。
- 能说清 VRS 不是改写刚体动力学，而是修正旋翼入流和有效拉力。
- 能列出初始验证场景和数据缺口。

## 3. 阶段二：模型规格冻结

目标是回答“后面实现时，每个变量、接口和模块到底是什么意思”。

阶段二建议按这个顺序走：

1. 轴系与符号冻结：以 [轴系与符号冻结约定](axis-conventions.md) 为准，统一坐标系、姿态角、速度、角速度、力矩和下降率正方向。
2. 状态向量冻结：定义状态 $x$ 的分量、单位、正方向和初始值来源。
3. 输入向量冻结：定义操纵输入 $u$，包括总距、纵向周期变距、横向周期变距和尾桨总距。
4. 六自由度动力学规格：写清 $\dot{x}=f(x,u,t)$ 中刚体动力学部分怎么接收合力和合力矩。
5. 外力外矩模块规格：定义主旋翼、尾桨、机身、平尾、垂尾和重力的接口。
6. VRS 接口规格：定义 `VRS_Detection` 和 `VRS_Correction` 的输入、输出和触发逻辑。
7. 验证路线冻结：以 [验证路线图](verification-roadmap.md) 为准，确定每个场景的输入、观测量和通过标准。

阶段二的完成标准：

- `docs/stage-2-model-spec-draft.md` 从“草案”变成“可执行规格”。
- 所有符号、单位、正方向和接口都能被后续代码直接引用。
- 没有“同一个变量在不同文档里含义不同”的情况。

## 4. 阶段三：最小仿真实现

目标是先做一个能跑通 VRS 进入和改出的最小模型。

第一版不追求高保真，建议只实现：

- `RigidBody6DOF`：六自由度主体。
- `MainRotor`：简化主旋翼力和力矩。
- `InflowModel`：均匀入流。
- `VRS_Detection`：工程判据。
- `VRS_Correction`：诱导速度或有效拉力修正。
- `ScenarioRunner`：悬停、正常下降、VRS 进入、改出、参数扫掠。

第一版状态方程仍保持：

$$
\dot{x}=f(x,u,t)
$$

阶段三的完成标准：

- 能从初始条件跑出时间响应。
- 能输出 $V_h$、$V_d$、$v_i$、$T_\mathrm{nom}$、$T_\mathrm{eff}$、$s_\mathrm{VRS}$ 等信号。
- 正常下降不触发 VRS，低速大下降率能触发 VRS。

## 5. 阶段四：验证与调参

目标是判断模型是否符合物理趋势，而不是只看程序能不能运行。

验证顺序：

1. 悬停：检查 $T \approx W$。
2. 正常下降：检查 $s_\mathrm{VRS}=0$。
3. VRS 进入：检查 $s_\mathrm{VRS}>0$，并出现 $T_\mathrm{eff}<T_\mathrm{nom}$ 或下沉增强。
4. 改出：检查 $s_\mathrm{VRS}$ 回落，垂向速度恢复。
5. 参数扫掠：得到随 $V_h$、$V_d$ 和总距变化的危险区边界。

阶段四的完成标准：

- 每个验证场景都有输入、输出图或表格、结论。
- 能解释模型失败时是参数问题、判据问题、入流问题还是动力学接口问题。
- 能形成下一轮模型修正清单。

## 6. 阶段五：模型升级

只有在最小模型跑通后才进入这一阶段。可选升级方向包括：

- 将均匀入流升级为 Pitt-Peters 动态入流。
- 将准定常挥舞升级为一阶挥舞动力学。
- 加入主旋翼下洗对尾桨、平尾和垂尾的影响。
- 用翼型极曲线替换线性升力线。
- 引入公开试飞或风洞数据做更严格对比。

阶段五的原则是：每次只升级一个模块，并用阶段四的验证场景证明升级确实改善了模型。
